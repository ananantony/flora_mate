1. 设计原则

1. **简单优先（KISS）**：项目本身是"一次性顺序任务"，状态机模型最匹配——选择裸机而非 RTOS（决策依据见 § 13 附录 A）。
2. **Fail-Safe 默认安全态**：任何上电、复位、异常情形下，**CH1 水泵总电源**默认释放，水泵物理失电。"什么都没运行 = 一定不漏水"。
3. **强弱电隔离贯穿到代码**：操作 GPIO 输出脏侧负载（继电器、PWM）的函数必须独立模块，且**永远先关后开**——执行下一个动作前先把上一个动作回零。
4. **可观测性优先**：每个状态切换、每次按键、每次 I²C 读写都要有 USART1 日志输出，方便现场诊断；OLED 显示与日志双通道反映系统内部状态。
5. **配置与代码分离**：所有"可调参数"（每路时长、PWM 占空比、阶梯数、按键长短按门限）统一存 AT24C64，代码中只放默认值与硬限边界。
6. **状态转换显式化**：状态机每个 transition 必须由唯一事件触发；禁止"侧门"（如某 ISR 直接修改主状态变量）。

2. 软件总体架构

### 2.1 分层

```
 ┌─────────────────────────────────────────────────────────────┐
 │                APP 层（应用业务）                            │
 │   app_main_fsm   ─ 主状态机（系统级）                       │
 │   app_pump_fsm   ─ 单路浇灌子状态机（PWM 阶梯）             │
 │   app_menu       ─ 调试 / 参数菜单导航                       │
 │   app_display    ─ OLED 页面绘制（IDLE / RUN / MENU / ERR）  │
│   app_serial_debug ─ USART1 串口调试命令与调试状态入口        │
 │   app_event      ─ 事件队列（按键 / 超时 / 故障）            │
 ├─────────────────────────────────────────────────────────────┤
 │                BSP 层（板级抽象）                            │
 │   bsp_relay      ─ 6 路继电器（CH1 泵电源互锁；CH2~5 阀）    │
 │   bsp_pump_pwm   ─ 水泵 PWM（占空比 0~100% 安全设定）        │
 │   bsp_key        ─ 4 路独立按键（按下低电平）+ 去抖          │
 │   bsp_oled       ─ 1.3" 128×64 OLED I²C 驱动 + 局部刷新     │
 │   bsp_eeprom     ─ AT24C64 I²C 读写 + 配置区双 bank 保护     │
 │   bsp_uart_log   ─ USART1 printf 重定向 + RX 环形缓冲        │
 │   bsp_tick       ─ SysTick 1ms 时基 + 软定时器服务            │
 ├─────────────────────────────────────────────────────────────┤
 │                HAL 层（ST 官方 HAL，CubeMX 生成）            │
 └─────────────────────────────────────────────────────────────┘
```

### 2.2 时基与主循环

**SysTick = 1 ms**（HAL 默认）。所有"软定时"通过 32-bit 滴答计数器 `g_tick_ms` 累加，软件需要"超时 / 周期"时记下起点 tick，再用 `(now - start) >= duration` 判断。

主循环（`main()` 中的 `while(1)`）按"协作式分时"运行，每个 task 是非阻塞函数：

```
while (1) {
    bsp_tick_service();        // 1ms 必做：维护软定时器
    app_serial_debug_tick();   // 串口心跳、命令行解析
    if (app_main_fsm_key_input_enabled()) {
        bsp_key_scan();        // 串口调试态下暂停按键扫描
        app_event_dispatch();  // 处理事件队列
    }
    app_main_fsm_step();       // 主状态机推进一步
    bsp_uart_log_flush();      // 把日志环形缓冲输出
    app_display_refresh();     // 100ms 内部限速；局部刷新
}
```

> 每个任务函数**内部自带"距上次执行时间"判断**，到点才真正干活，否则立即返回。这是裸机替代 RTOS 多任务的标准做法（即"软任务"）。

### 2.3 中断使用

| 中断 | 用途 | ISR 工作 |
|---|---|---|
| SysTick | 1ms 时基 | 仅 `g_tick_ms++`（HAL 自动） |
| USART1 | RX 收命令字节 | 仅"收 1 字节 → 写入环形缓冲 → 重启接收"，**不解析、不打印** |

所有其他中断（TIM2/I2C1/EXTI/IWDG…）**全部禁用**。

3. 主状态机（System FSM）

### 3.1 状态枚举

```
SYS_SERIAL_WAIT     上电后等待串口调试入口命令；本阶段不自动浇灌
SYS_SERIAL_DEBUG    串口调试状态机；OLED 显示 Serial Debug，按键响应暂停
SYS_BOOT            预留：上电延时 500ms，外设初始化
SYS_SELFTEST        读 EEPROM 配置；I²C 总线扫描；继电器逐路咔嗒自检
SYS_IDLE_3S         OLED 倒计时 3 秒，等待用户决定
SYS_AUTO_RUN        自动浇灌主任务（CH1 水泵电源吸合，进入 PUMP_FSM）
SYS_MENU            调试 / 参数菜单（手动测试 + 改参数）
SYS_DONE            任务完成，关 CH1，显示"完成"
SYS_ERROR           异常态，关全部继电器，红灯急闪，OLED 显示 ERR 码
SYS_SLEEP           主任务结束后空闲等待米家断电（**不进低功耗模式**）
```

### 3.2 状态转换图

```
       ┌─────────────────────────────────────────────────────┐
       │                                                     │
       │                      [ANY] → SYS_ERROR (超时/水泵堵转/I²C 失联) │
       │                                                     │
   POWER_ON                                                  ▼
       │
       ▼
   SYS_SERIAL_WAIT ── 收到 debug/debug on/dbg ──→ SYS_SERIAL_DEBUG
   SYS_SERIAL_WAIT ── 超时 3s 无 debug ──→ SYS_BOOT → … → AUTO_RUN
       │
       │ 本阶段不自动进入 SYS_AUTO_RUN；未收到命令则保持安全等待
       │
       ▼（后续版本恢复自动流程时使用）
   SYS_BOOT → SYS_SELFTEST
       │ 自检通过
       ▼
   SYS_IDLE_3S ─── 3s 内任意键按下 ──→ SYS_MENU
       │
       │ 3s 内无按键 / 或 K1 跳过
       ▼
   SYS_AUTO_RUN ─── K3 (停止) ─────→ SYS_DONE
       │
       │ 4 路全部完成
       ▼
   SYS_DONE
       │
       ▼
   SYS_SLEEP（OLED 显示"完成，等待断电"，永久驻留直到米家插座断电）

   SYS_MENU ─── 退出菜单（K3 长按 / 选择"开始浇水"） ──→ SYS_AUTO_RUN
   SYS_MENU ─── 选择"退出待机" ────────────────────→ SYS_SLEEP
```

### 3.3 上电后的串口调试等待行为

- 上电完成 BSP/APP 初始化后，主状态机进入 `SYS_SERIAL_WAIT`（默认 3s，宏 `APP_SERIAL_DEBUG_BOOT_WAIT_MS`），继电器全释放、PWM 为 0。
- 等待窗内 USART1 每秒打印心跳：`[I] hb uptime_ms=... wait_remain_ms=...`。
- 若收到 `debug` / `debug on` / `dbg`，进入 `SYS_SERIAL_DEBUG`；OLED 显示当前命令与 OK/ERR；**调试态不再打印心跳**；按键扫描暂停；`valve` 经 `Bsp_Relay_DebugSet` 无互锁（1~6 可多路同时 on）。
- 若 3s 内未收到 debug，超时进入 `SYS_BOOT → SELFTEST → IDLE_3S → AUTO_RUN` 正常量产流程。
- 编译宏 `APP_SERIAL_DEBUG_AUTO_ENTER_ON_BOOT=1` 时跳过等待窗，上电直接进入 `SYS_SERIAL_DEBUG`（见 `app_serial_debug_config.h`）。

### 3.4 原上电"3 秒倒计时"行为（量产流程预留）

- 进入 `SYS_IDLE_3S` 后，OLED 显示大字：`3 → 2 → 1 → 启动`
- 倒计时期间任意 K1/K2/K3/K4 按下都进入 `SYS_MENU`（最大化"误碰"也能进调试，避免直接执行不想要的任务）
- 倒计时归零自动进入 `SYS_AUTO_RUN`
- 倒计时长度（默认 3s）从 EEPROM 读取，菜单可调 1~30s

### 3.5 SYS_AUTO_RUN 中的"停止按钮"语义

- **K3 = 停止键**（与总设 § 3.2 一致，但本版强化为"硬退出"）：
  - 按下后立即关 PWM、关全部水阀、释放 CH1
  - 状态机跳到 `SYS_DONE`（不是回到 `SYS_IDLE_3S`，避免误启）
  - OLED 显示"用户中止，等待断电"
- **K1 = 跳过当前路**（次要功能，可选实现）
- **K2 = 暂停 / 恢复**（次要功能，可选实现）
- **K4 = 进入运行时菜单**（仅在路间静默期生效，工作期内忽略）

4. 单路浇灌子状态机（Pump FSM）

### 4.1 调用关系

主状态机 `SYS_AUTO_RUN` 中以"通道号 i = 1..4"为参数依次调用 `app_pump_fsm_run(channel_i)`，每路一个完整生命周期约 30~120 秒。

### 4.2 子状态枚举

```
PUMP_INIT           吸合阀 CH(2+i) → 等阀机械到位 200ms
PUMP_OPEN_MAIN      吸合 CH1（水泵总电源）→ 等待稳压 100ms
PUMP_STEP_1         PWM 占空比 = step1 (默认 35%)，持续 t1 秒
PUMP_STEP_2         PWM 占空比 = step2 (默认 55%)，持续 t2 秒
PUMP_STEP_3         PWM 占空比 = step3 (默认 75%)，持续 t3 秒
PUMP_STEP_4         PWM 占空比 = step4 (默认 95%)，持续 t4 秒
PUMP_RAMP_DOWN      PWM 占空比线性降到 0（避免水锤），耗时 500ms
PUMP_CLOSE_MAIN     释放 CH1（断水泵供电）→ 卸压等待 500ms
PUMP_CLOSE_VALVE    释放 CH_i
PUMP_GAP            路间静默 500ms（让水路压力归零）
PUMP_DONE           本路完成，返回主状态机
```

### 4.3 PWM 阶梯调速的含义

地插喷头是**被动机械喷头**，**出水半径 ∝ 出水压力 ∝ 水泵转速 ∝ PWM 占空比**。

| 档位 | 默认占空比 | 默认持续时长 | 喷射半径效果 |
|---|---|---|---|
| 1 档 | 35% | 3 s | 近距离淋洒（盆缘内圈） |
| 2 档 | 55% | 3 s | 覆盖盆中区 |
| 3 档 | 75% | 3 s | 覆盖盆远端 |
| 4 档 | 95% | 3 s | 满压（盆外缘） |

总单路时长 = 4 × 3 + ramp_down 0.5 + 各状态间隔 ≈ 13.5 秒；4 路总任务约 1 分钟（含路间静默）。

### 4.4 关键时序保证

- **CH1 与阀 CH2~CH5 的吸合次序**：先吸合当前路阀，再吸合 CH1 水泵总电源；先释放 CH1，再释放阀。**避免水锤与空抽**。
- **PWM 占空比修改时机**：仅在 STEP_x 进入时设一次 CCR；中途不调。
- **每档结束的 ramp 是否需要？** 不需要——4 档之间直接跳变没问题，因为水泵机械惯性 + 水路压力惯性已经能"软化"过渡；只在最末档结束时做 ramp_down 防止关泵水锤。

### 4.5 异常分支

| 触发 | 处理 |
|---|---|
| 单路超时（默认 90s，硬限 120s） | 立即进 `PUMP_RAMP_DOWN` → `PUMP_CLOSE_MAIN` → `PUMP_CLOSE_VALVE`，主状态机置 `SYS_ERROR` |
| K3 停止键 | 同上 |
| I²C 失联（OLED / EEPROM 写失败 3 次） | 仅记录日志，不中断浇水；任务完成后置 ERROR 显示 |

5. 输入子系统（按键）

### 5.1 物理与电气

| 按键 | GPIO | 默认状态 | 按下状态 |
|---|---|---|---|
| K1 | PA1 | 高（内部上拉） | 低 |
| K2 | PA2 | 高 | 低 |
| K3 | PA3 | 高 | 低 |
| K4 | PA4 | 高 | 低 |

### 5.2 扫描算法（去抖 + 长短按）

每 **20 ms** 扫描一次（在主循环中 `bsp_key_scan` 内部限速）：

1. 读 4 路 GPIO 当前电平 → 反相 → 当前状态 `cur`（bit0~bit3 表示 K1~K4 是否按下）；
2. 去抖：若 `cur == last`，去抖计数器 `stable_count++`；否则归零。`stable_count >= 3`（即连续 60ms 不变）则采用为"稳定按下/释放"；
3. **短按** = 稳定按下 → 稳定释放，且按下时长 < 1s，释放瞬间产生 `KEY_EVT_SHORT_K_x`；
4. **长按** = 稳定按下持续 >= 1s，**按下时长 = 1s 的瞬间**产生 `KEY_EVT_LONG_K_x`（不等释放）；
5. **超长按** = >= 3s，产生 `KEY_EVT_HOLD_K_x`（用于"长按 K3 强制复位"等保命操作）。

### 5.3 事件分发

按键事件**不直接调用业务函数**，而是 `app_event_post(KEY_EVT_xxx)` 投递到一个 8 槽位的事件队列，由 `app_event_dispatch` 在主循环中统一消费。这样保证：
- 状态机切换是"同步"的，不会被 ISR 中断切到一半；
- 同一时刻多个按键事件不冲突；
- 调试时可以打印事件队列查看时序。

### 5.4 按键功能映射（按状态而异）

| 状态 | K1 | K2 | K3 | K4 |
|---|---|---|---|---|
| SYS_SERIAL_WAIT / SYS_SERIAL_DEBUG | 无效（暂停扫描） | 无效（暂停扫描） | 无效（暂停扫描） | 无效（暂停扫描） |
| SYS_IDLE_3S | 跳过倒计时立即启动 | 进菜单 | 进菜单 | 进菜单 |
| SYS_AUTO_RUN | 跳过当前路 | 暂停 / 恢复 | **停止整个任务** | 进运行时菜单 |
| SYS_MENU | 上 / 减 | 下 / 加 | 返回上层 / 长按退出菜单 | 确认 / 进入子项 |
| SYS_DONE / SYS_SLEEP | 无效 | 无效 | 长按 3s 强制重启 | 无效 |
| SYS_ERROR | 无效 | 无效 | 长按 3s 强制重启 | 切错误码翻页 |

6. 输出执行子系统

### 6.1 继电器封装（`bsp_relay`）

封装层提供**逻辑通道号**而非物理 GPIO，对上层屏蔽极性（高电平触发）：

```
bsp_relay_set(BSP_RELAY_VALVE_1, ON);       // CH2 阀1，PB13
bsp_relay_set(BSP_RELAY_PUMP_PWR_CH1, OFF); // CH1 泵电源，PB12
bsp_relay_all_off();                      // 全部置 LOW (释放) -- Fail-Safe 状态
```

**关键互锁**：
- 提供 `bsp_relay_main_off_force()`，**任何路径下都可调用**，无任何前置条件；这是异常处理中调用的"硬刹车"。
- `bsp_relay_set(PUMP_PWR_CH1, ON)` 须至少一路阀 CH2~CH5 已开，否则拒绝（防憋压）。

### 6.2 水泵 PWM 封装（`bsp_pump_pwm`）

```
bsp_pump_pwm_init();              // 启动 TIM2 PWM，CCR = 0
bsp_pump_pwm_set_duty(uint8_t d); // d = 0~100，限幅 0~95（不允许长期 100%）
bsp_pump_pwm_stop();              // CCR = 0，停泵
```

**安全策略**：
- 限幅 0 ~ **95**（绝对硬限），防止 100% 占空比下 LR7843 持续直通发热；
- `set_duty(d>0)` 时若 `bsp_relay_get(PUMP_PWR_CH1) == OFF`，**自动拒绝并打 ERROR 日志**；
- 提供 `bsp_pump_pwm_ramp_down(uint32_t ms)` 在指定时间内线性把占空比降到 0。

7. 显示子系统（OLED）

### 7.1 驱动

`bsp_oled` 直接复用厂家 SSD1315 例程的核心函数（与 SSD1306 兼容），提供：

```
bsp_oled_init();
bsp_oled_clear();
bsp_oled_show_string(uint8_t x, uint8_t y, const char *s);

bsp_oled_fill_rect(uint8_t x, uint8_t y, uint8_t w, uint8_t h, bool on);
bsp_oled_flush();                 // 把 RAM frame buffer 推给芯片

```

### 7.2 页面定义（`app_display`）

| 页面 ID | 触发状态 | 内容 |
|---|---|---|
| `DISP_SERIAL_WAIT` | SYS_SERIAL_WAIT | "Serial Wait" + `debug in Ns` 倒计时 + `Send: debug` |
| `DISP_SERIAL_DEBUG` | SYS_SERIAL_DEBUG | 当前命令、`OK/ERR`、PWM、6 路继电器位图 |
| `DISP_SERIAL_DEBUG` | SYS_SERIAL_DEBUG | "Serial Debug" + "Keys disabled" + `Type help in UART` |
| `DISP_BOOT` | SYS_BOOT / SELFTEST | "FloraMate V1.0" + 进度条 |
| `DISP_IDLE_3S` | SYS_IDLE_3S | 大字倒计时 "3 → 2 → 1"；底栏"按任意键进菜单" |
| `DISP_RUN` | SYS_AUTO_RUN | 第 1 行：CHx i/4；第 2 行：阶梯进度（▰▰▰▱）；第 3 行：占空比 75%；第 4 行：剩余 12s |
| `DISP_DONE` | SYS_DONE | 大字"完成"+ 已用时长 |
| `DISP_MENU_MAIN` | SYS_MENU | 菜单条目（▶ 当前项）|
| `DISP_MENU_PARAM_EDIT` | 编辑参数子页 | 显示参数名 + 当前值 + ▲▼ |
| `DISP_MENU_MANUAL_TEST` | 手动测试子页 | 4 阀 + 水泵的逐路点击测试 |
| `DISP_ERROR` | SYS_ERROR | "ERR-xxx" + 简要描述 |

### 7.3 刷新策略

- **全屏刷新仅在页面切换时**做一次（~10 ms）；
- **页面内动态区**（如倒计时数字、PWM 占空比）用**局部刷新**——只把变化的那 8×16 像素区域推给芯片（~1 ms）；
- 主循环中 `app_display_refresh` **内部限速到 100 ms**，避免 I²C 占满总线；
- **烧屏防护**：在 RUN 状态下每分钟把整个显示内容 X 坐标偏移 ±2 像素一次（OLED 烧屏主要由"长期固定位置 + 高亮像素"引起，偏移 2px 已足够延寿）。

8. 日志子系统（USART1）

### 8.1 输出（TX）

- 在 `syscalls.c` 重定向 `_write` / `fputc` 到 `bsp_uart_log_write_bytes`；
- 上层用法：`LOGI("auto run: channel %d, duty=%d%%", ch, duty);`
- 宏定义 4 个等级：`LOGE` 错误 / `LOGW` 警告 / `LOGI` 信息 / `LOGD` 调试；
- 等级阈值在 EEPROM 配置中可调（默认 LOGI 以上输出）；
- **输出是异步的**：`LOGx` 只把字符串拷贝到 256B 环形发送缓冲，主循环中 `bsp_uart_log_flush` 通过 `HAL_UART_Transmit_IT` 非阻塞推送；避免任何 LOG 调用阻塞状态机。

### 8.2 输入（RX）与串口调试命令

- USART1 RX 中断 → 收 1 字节 → 写入 64B 环形接收缓冲 → 重启接收；环满记录 `rx_drop`，UART 错误记录 `rx_err`。
- RX 使用 `HAL_UARTEx_ReceiveToIdle_IT`（无 DMA）；主循环 `TryGetLine` 遇 `\r` / `\n` 成行，无换行时由 **USART IDLE** 帧结束提交一行。
- 上电默认进入 `SYS_SERIAL_WAIT`（3s）。收到 `debug` / `debug on` / `dbg` 后进入 `SYS_SERIAL_DEBUG`；超时无 debug 则进入正常 BOOT 流程。`APP_SERIAL_DEBUG_AUTO_ENTER_ON_BOOT=1` 可跳过等待窗。
- 等待窗内每秒心跳；`SYS_SERIAL_DEBUG` 下不打印心跳。
- 命令集以 `doc/串口调试指令说明_V1.0.md` 为单点说明，当前实现包括：
  - `help` / `?`、`ver`、`status`
  - `pump <0-100>` / `pump off`
  - `valve open <1-6>` / `valve close [1-6]`（调试路径 `Bsp_Relay_DebugSet`，无阀互锁，可多路同时 on）
  - `cfg dump|get|set|save|load|reset`
  - `stop`、`reset`
  - `run` 仅保留，不启动自动浇灌
- `pump` 经 `Bsp_Relay_Set(CH1)`，受阀互锁约束；`valve` 调试命令绕过互锁（编号 1=CH1 泵，2~5=阀，6=备用）。

9. 参数存储子系统（AT24C64）

### 9.1 EEPROM 物理布局

| 起始 | 长度 | 区域 |
|---|---|---|
| 0x0000 | 64 B | Bank A 头：magic(4)+version(2)+CRC16(2)+payload_size(2)+保留(54) |
| 0x0040 | 256 B | Bank A payload（用户配置） |
| 0x0140 | 64 B | Bank B 头（结构同 A） |
| 0x0180 | 256 B | Bank B payload（A 的冗余副本） |
| 0x0280 | — | 预留 |

### 9.2 双 Bank 写入策略

为防止"写入过程中突然断电（米家插座硬切电）"导致配置损坏：

1. **读取**：先校验 Bank A 头部 CRC16，OK 则取 A；否则校验 Bank B；都失败则使用代码内置的工厂默认参数，**并打日志告警**；
2. **写入**：先写入 **当前未启用的 Bank**（A 在用则写 B，反之），写完再校验，再"切换标志"。即使写入过程中掉电，下次启动用旧 Bank 仍可工作。

### 9.3 配置数据结构

```c
typedef struct {
    uint16_t magic;          // 0xFAC3
    uint8_t  version;        // 当前 1
    uint8_t  reserved;
    /* 浇灌参数 */
    uint8_t  step_count;     // 阶梯档数 1~8，默认 4
    uint8_t  step_duty[8];   // 每档占空比 0~95，默认 35/55/75/95
    uint8_t  step_seconds[8];// 每档持续秒数，默认 3
    uint8_t  channel_enable; // bit0~3：4 路开关，默认 0x0F
    /* 时序参数 */
    uint8_t  idle_seconds;       // 倒计时长度，默认 3
    uint8_t  inter_gap_ms_x10;   // 路间静默 × 10ms，默认 50（500ms）
    uint16_t per_channel_timeout_s;  // 单路超时秒，默认 90
    uint16_t total_timeout_s;        // 总任务超时秒，默认 480
    /* HMI 参数 */
    uint8_t  long_press_ms_x10;  // 长按门限 × 10ms，默认 100（1s）
    uint8_t  log_level;          // 0~3：LOGE / LOGW / LOGI / LOGD
    /* 末尾 */
    uint16_t crc16;          // 全结构 CRC（不含自身）
} app_config_t;
```

> 总长 < 64 字节，留有充裕扩展空间。

### 9.4 默认值与工厂复位

- 上电校验 EEPROM 失败 → 加载代码内 `g_factory_config` → **打印 WARN 日志** + OLED 显示"⚠ 已加载默认参数"
- 菜单中提供"恢复出厂"项，长按 K4 3 秒确认，写入两个 Bank 完成复位

### 9.5 详细字段定义

本节是概览。**`app_config_t` 的完整字段定义、AT24C64 物理地址映射、双 Bank 启动加载/写入流程、CRC 算法、菜单与串口命令映射、版本演进策略、EEPROM 寿命核算、验收测试**等内容统一收录于：

10. SYS_MENU 调试模式（用户核心需求）

### 10.1 进入条件

- `SYS_IDLE_3S` 状态下任意键按下即进入 `SYS_MENU`；
- `SYS_AUTO_RUN` 状态下 K4 按下，**在路间静默期**才接受切入 `SYS_MENU`（路工作中按 K4 被忽略，避免误触）。

### 10.2 菜单结构

```
[主菜单]
  ├─ 1. 开始浇水            → 退出菜单，直接进入 SYS_AUTO_RUN
  ├─ 2. 参数设置
  │     ├─ 阶梯档数
  │     ├─ 各档占空比
  │     ├─ 各档持续秒
  │     ├─ 启用通道（bit0~3）
  │     ├─ 倒计时长度
  │     ├─ 单路超时
  │     ├─ 日志等级
  │     ├─ ...
  │     └─ 保存到 EEPROM
  ├─ 3. 手动测试
  │     ├─ 阀 1~4 开 / 关       （CH2~CH5）
  │     ├─ 水泵电源 CH1 开 / 关
  │     ├─ CH6 备用
  │     └─ 水泵 PWM 0~95%
  ├─ 4. 系统信息            → 显示固件版本、I²C 设备扫描、累计运行时长、最近错误
  ├─ 5. 恢复出厂参数（需长按 K4 3s 确认）
  └─ 6. 退出（不浇水）       → SYS_SLEEP
```

### 10.3 手动测试的安全规则

- **手动测试模式 = 完全旁路自动状态机**，但仍受 BSP 层的硬互锁保护：
  - PWM>0 时须 CH1 已吸合；
  - 须先开至少一路阀再开 CH1，顺序错误 OLED 报警；
- 退出菜单时**自动关闭所有继电器与 PWM**，恢复 Fail-Safe 默认态；
- 菜单中所有 GPIO 操作都打 LOGI 日志。

11. 异常与安全

### 11.1 异常分类

| 错误码 | 名称 | 触发条件 | 处理动作 |
|---|---|---|---|
| ERR_001 | EEPROM 校验失败 | 两 Bank CRC 均错 | 加载工厂值，继续运行，OLED 显示 ⚠ |
| ERR_002 | I²C OLED 失联 | 连续 3 次写超时 | 忽略 OLED，继续浇水，LOGE 日志 |
| ERR_003 | I²C EEPROM 失联 | 连续 3 次读失败 | 同上 |
| ERR_010 | 单路浇灌超时 | 持续时间 > per_channel_timeout_s | 立即关泵关阀，进 SYS_ERROR |
| ERR_011 | 总任务超时 | 总时长 > total_timeout_s | 同上 |
| ERR_012 | 互锁失败 | CH1=ON 但无阀开 | 拒绝执行，LOGE |
| ERR_013 | 水泵 PWM 误设 | duty>0 但 CH1=OFF | 同上 |
| ERR_020 | 看门狗复位 | 主循环阻塞（本版不启用 IWDG，预留） | — |
| ERR_FF | 未知异常 | HardFault / NMI | 显示 ERR-FF，等待断电 |

### 11.2 SYS_ERROR 状态行为

- `bsp_relay_all_off()` 立即调用，**所有继电器物理释放**；
- PWM CCR 写 0；
- OLED 显示 ERR 码 + 错误描述 + "请断电重启"；
- 板载 PC13 LED 100ms 急闪（区别于正常态 500ms 慢闪）；
- 等待米家插座自然断电，**MCU 不主动复位**（避免无限重启循环造成对水泵的反复抽搐）。

### 11.3 看门狗策略

V1.0 软件**暂不启用 IWDG**（与老 F103 版一致，与硬件方案 § 6.3 / CubeMX § 6.2 一致）。理由：
1. 开发调试期间断点会让 IWDG 误超时复位；
2. 本机有 CH1 水泵电源硬切断 + 米家插座两道物理安全门；
3. 量产固化时（Phase 2）再评估是否启用 IWDG（1 秒周期，主循环尾喂狗）。

12. 关键时序图（与用户描述精确对照）

### 12.1 上电 → 串口等待 / 调试 / 正常流程

```
t=0      米家插座通电 → MCU 上电
t<1s      APP 初始化 → SYS_SERIAL_WAIT（或 AUTO_ENTER 时直进 SYS_SERIAL_DEBUG）
t=0~3s    等待窗：每秒 hb + wait_remain_ms；OLED 显示 debug in Ns
分支 A    3s 内收到 debug → SYS_SERIAL_DEBUG（无 hb；OLED 显示命令/OK/ERR）
分支 B    3s 超时无 debug → SYS_BOOT → SELFTEST → IDLE_3S → AUTO_RUN
t=调试    valve open 1 → valve open 2（可同时 on）→ pump 30 → stop
```

### 12.2 K3 停止键中断流程

```
t=任意    用户按下 K3
t=+20ms   按键扫描捕获到稳定下沉
t=+20ms   投递事件 KEY_EVT_SHORT_K3 到队列
t=+21ms   app_event_dispatch 取出事件
t=+21ms   主状态机接收 → 立即调用 bsp_relay_main_off_force()
t=+22ms   PWM CCR=0；CH1 释放；4 路阀全释放
t=+22ms   状态机跳到 SYS_DONE，OLED 显示"用户中止"
t=+...    等待断电
```

整个流程**< 100ms 完成全部物理切断**（与 V1.2 总设 § 8 关键技术指标一致）。

13. 附录

### A. 为什么不用 FreeRTOS？

| 维度 | 评估 |
|---|---|
| 项目执行模型 | 一次性顺序任务，状态机天然匹配 |
| 实时性 | 全部软实时（按键 20ms，OLED 100ms，停止 100ms），裸机够用 |
| PWM | 硬件 TIM2 自动产生，无需 CPU 干预 |
| 已有代码 | 老 F103 版裸机状态机已验证可靠，可大段移植 |
| 调试 | 单线程主循环 + 状态变量，HardFault 排查最简单 |
| 资源 | F401 资源富余，但裸机省下的复杂度 > 节约的 RAM |
| 扩展性 | Phase 2 / 3 加联网时再上 RTOS 不迟 |

### B. 与老 F103 版本软件设计文档的差异

| 维度 | 老 F103 版 | 本 F401 版 V1.0 |
|---|---|---|
| 主控 | F103C8T6 / 72MHz | F401RCT6 / 84MHz |
| 操作系统 | 裸机 | **裸机（保持一致）** |
| HMI | 无 OLED / 按键 | **加 4 按键 + OLED 菜单 + 手动测试** |
| 存储 | 内部 Flash EEPROM emulation | **AT24C64 双 Bank** |
| 主任务 | 通过串口命令触发 / 米家定时 | **米家定时上电 + 3s 倒计时自动启动** |
| 停止 | 串口 `stop` 命令 | **K3 物理按键 + 串口都支持** |
| PWM | 10 kHz | **100 Hz**（当前调速参数） |
| 水泵总电源 | CH5 总闸+浮球 | **CH1** 仅 MCU（浮球 Phase 2 串 CH1 线圈） |
| 子模块结构 | 单文件 main + 几个驱动 | **BSP / APP 分层（§ 2.1）** |
| 日志 | USART1 printf | **加事件级别 + 异步环形缓冲** |

### C. 待定 / Phase 2 议题

1. 启用 IWDG 看门狗（量产固化前评估）；
2. 启用浮球串入 CH1 线圈回路（拆 LO_LEVEL_SHORT）；
3. 接入 AJ-SR04M 超声波液位（PA8 占用，软件加 `bsp_ultrasonic`）；
4. 加水浸传感器（PB10），触发即软件强制关 CH1；
5. OLED 字模如果超过 4 KB（中文显示），考虑放进 MCU 内部 Flash 的常量段，无需外置存储。
