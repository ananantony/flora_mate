2. 总体硬件框图 (System Block Diagram)

系统在物理上由"脏电动力侧"与"纯净控制侧"两半构成，仅通过 B1205S-2WR3 隔离模块的"磁"与继电器/光耦的"光"耦合。两半的 GND 在物理世界中永不相交。

```
                          ┌────────────── 米家 220V AC 智能插座 (L0) ───────────────┐
                          │ 仅在浇灌窗口（约 5~10 min/日）通电                       │
                          └──────────────────┬────────────────────────────────────┘
                                             │
                                       12V/4A 适配器
                                             │
                       ┌─────────────────────┴────────────────┐
                       │  电源入口：NTC 5D-9 + 470µF + SMBJ15CA │   ← 浪涌三件套
                       └─────────────────────┬────────────────┘
                                             │
                              ┌──────────────┴───────────────────┐
                              │ 12V_DIRTY 母线（动力网，UL1007 20AWG）│
                              └──┬───────────┬───────────┬───────┘
                                 │           │           │
                  ┌──────────────▼┐   ┌──────▼──────┐   ┌▼──────────────────┐
                  │ LM2596 降压模块 │   │ B1205S-2WR3 │   │ 6 路继电器 模块 DC+ │
                  │ 12V → 5V_DIRTY  │   │ 12V → 5V_ISO│   │ （继电器线圈 5V_DIRTY）│
                  └────────┬───────┘   └──────┬──────┘   └──┬─┬─┬─┬─┬─┬───┘
                           │                  │             │ │ │ │ │ │
                           └→ 继电器线圈 公用  └→ AMS1117 →   │ │ │ │ │ └─ CH6 预留
                                              3.3V (MCU)    │ │ │ │ └── CH5 安全总闸 ──┐
                                                            │ │ │ └────── CH4 区阀 4   │
                                                            │ │ └──────── CH3 区阀 3   │
                                                            │ └────────── CH2 区阀 2   │
                                                            └──────────── CH1 区阀 1   │
                                                                                       │
                       ┌───────────────────────────────────────────────────────────────┘
                       │ 12V_DIRTY (经 CH5 NO 触点) → LR7843 MOS 调速板 → 水泵
                       │                                       ↑
                       │                                       │ PA0 PWM 25 kHz (隔离前)
                       │                                  PC817 光耦
                       └─ MCU GPIO → 6 路继电器 IN（低电平触发，3.3V 直驱）
                                  → I²C (PB6/PB7) → SSD1315 OLED + AT24C64
                                  → 4× GPIO (PA1~PA4) → K1/K2/K3/K4 按键
                                  → PC13 → 板载心跳 LED
```

要点：
  · **CH5 不是 4 路阀中的某一路**，是 **MCU GPIO 与浮球开关串联** 控制的"水泵-12V 通断 总闸"。其 NO 触点串入水泵 +12V 的进线；任意一路阀工作期间 CH5 必须吸合，否则水泵不出水。
  · **CH6 预留**，在 Phase 2 时用于水浸/超声波等扩展。
  · MCU 与所有功率器件**严格双地隔离**：MCU 侧地为 `GND_ISO`（B1205S 二次侧），动力侧地为 `GND_DIRTY`。两者**仅在 B1205S 内部初次级耦合**存在能量通路。

3. 三级断电架构 (Three-Layer Cutoff)

物理串联三道开关，任意一道开路均切断水泵动力：

| 级 | 名称 | 控制方 | 触发条件 | 失效安全态 |
|---|---|---|---|---|
| **L0** | 米家 220V 智能插座 | 用户 / 米家 APP 定时 | 浇灌窗口内闭合（约 5~10 min/日） | 开路（无市电） |
| **L1** | CH5 安全总闸继电器（NO 触点） | MCU GPIO **AND** 浮球开关 | MCU 命令吸合 **且** 浮球高位 | 开路（继电器释放） |
| **L2** | CH1~CH4 路阀继电器（NO 触点） | MCU GPIO（单一路） | 单路 PWM 阶梯期间吸合 | 开路（继电器释放） |

L1 中的"AND"由硬件实现：浮球开关（NC 常闭，水位足时闭合）直接**串联**进 CH5 继电器的 5V_DIRTY 线圈回路，**不经过 MCU GPIO**。即使 MCU 死机把 CH5 信号锁定为吸合，只要水箱低水位，CH5 线圈失电，水泵物理失电。

> 本版本**不安装浮球**：CH5 线圈回路中 PSF-202 的位置用一段**短接跳线**桥接（端子排上标注 `LO_LEVEL_SHORT`），Phase 2 拆掉跳线、串入浮球即可。**短接期间，L1 的"水位安全"功能由软件单独承担**，须在固件中显式注释。

6. 子模块详细接线 (Module Wiring)

6.1 6 路 5V 光耦隔离继电器模块

| 项 | 规格 |
|---|---|
| 型号 | 通用 6 路 5V 继电器模块（光耦隔离 + 800OP 三极管 + 5V 线圈继电器 + 跳线帽选择 H/L 触发） |
| 触发电平 | **低电平触发（COM-L / Active-Low）**，跳线帽必须打在 `L` 档 |
| 输入电平 | 3.3V/5V 双兼容（光耦输入端串 1kΩ 限流，3.3V 高电平 ≈ 释放，0V ≈ 吸合） |
| 控制端供电 | `VCC` 接 **5V_DIRTY**（来自 LM2596），`GND` 接 `GND_DIRTY`；**勿**接 MCU 的 3.3V/GND_ISO |
| 触点容量 | 单触点 10A/250VAC / 10A/30VDC（本项目仅 12V/0.5A 阀线圈与 12V/2~3A 水泵，余量充足） |
| **JD-VCC 跳线** | **拔掉跳线帽**，让继电器线圈 5V 与光耦输入侧 5V **物理隔离**；继电器线圈用 5V_DIRTY，光耦输入侧不接 VCC（详见下） |

> 关于 JD-VCC 跳线的处理 —— 这是本设计的关键工艺点：
> · 模块原厂状态：`JD-VCC` 与 `VCC` 通过跳线帽短接，意味着继电器线圈与光耦输入用同一组 5V，**等于没有隔离**。
> · 正确做法：**拔掉跳线帽**，`VCC` 不接（光耦输入侧自带 1kΩ 限流后直接由 MCU IN 引脚拉低驱动，光耦输出侧用 JD-VCC 单独供 5V_DIRTY），**这才是真正的光电隔离**。
> · 若发现拔掉跳线帽后继电器无法吸合，检查 `JD-VCC` 与 `GND_DIRTY` 是否给到 5V 与 0V，以及 `IN` 是否真正被拉到 < 1.2V。

通道分配（与 § 5 引脚表对齐）：

| 通道 | IN 接 MCU | NO 接 | COM 接 | 用途 | 续流二极管 |
|---|---|---|---|---|---|
| CH1 | PB12 | 阀 1 + 端 | 12V_DIRTY | 区阀 1 | 1N5822（贴阀本体） |
| CH2 | PB13 | 阀 2 + 端 | 12V_DIRTY | 区阀 2 | 1N5822 |
| CH3 | PB14 | 阀 3 + 端 | 12V_DIRTY | 区阀 3 | 1N5822 |
| CH4 | PB15 | 阀 4 + 端 | 12V_DIRTY | 区阀 4 | 1N5822 |
| CH5 | PB1 | LR7843 板 `12V+`（水泵动力输入） | 12V_DIRTY → **PSF-202 短接跳线** → CH5 COM | 安全总闸 | **SS54**（贴水泵端） |
| CH6 | PB0 | 空（端子台引出 1 脚） | 12V_DIRTY | Phase 2 预留 | — |

每路 NO-COM 触点跨接 **RC 灭弧网络**：0.1 µF/400V 陶瓷 + 100 Ω 1/4W 电阻 串联后并联在触点两端，降低拉弧与 EMI。

继电器 IN 端到 MCU GPIO 之间**不再额外串电阻**（模块板内已有 1kΩ）。

6.2 PC817 + LR7843 水泵调速模块

模块构成：
  · 光耦：PC817（输入侧 IF=10mA @ Uf≈1.2V，CTR≈100~200%）；
  · 主功率 MOS：LR7843（Vds=30V，Id=161A peak，Rds_on=2.4mΩ）；
  · 板载 1kΩ 光耦输入限流电阻 + 10kΩ 输出上拉 + 100kΩ MOS 栅极下拉；
  · 板载 SS54 续流二极管（保护 OUT± 端电感性负载）。

接线表：

| 模块端子 | 接至 | 说明 |
|---|---|---|
| `DC+` | **CH5 NO 触点输出**（即"经过 CH5 的 12V_DIRTY"） | 经总闸后的水泵动力正极 |
| `DC-` | `GND_DIRTY` 母线 | 公共地 |
| `OUT+` | 水泵 + 端（红线，UL1007 18~20 AWG） | 水泵正极 |
| `OUT-` | 水泵 - 端（黑线） | 水泵负极；与 `DC-` 在板内已并 |
| `PWM+` | **PA0**（TIM2_CH1） | **1 kHz** PWM 信号；3.3V 即可驱动 PC817 |
| `PWM-` | **GND_ISO**（MCU 侧 GND） | **重点**：信号地接 MCU 侧地，**勿**与 `DC-` 短接 |

> 隔离审核：PC817 输入侧用 `GND_ISO`，输出侧用 `GND_DIRTY`，光耦中间是物理空气隙 + 电气绝缘，确保两地完全隔离。**这是水泵调速实现强弱电分离的物理基础，不容妥协**。

PWM 极性：模块为"**高电平有效**"——PA0 输出高，水泵转；PA0 输出低，水泵停。固件中 PWM 默认占空比 0%，上电不起转。

> **关于 1 kHz 频率的工程权衡**：本项目采用 1 kHz 是基于 PC817 光耦的实际响应时间（典型 t_rise/t_fall ≈ 3~4 µs，1 kHz 周期 1000 µs，工作裕度极大；25 kHz 周期 40 µs，PC817 输出波形会变方有振铃）。**缺点**是 1 kHz 落在人耳最敏感频段（300 Hz ~ 3 kHz），水泵线圈会产生可闻的"嗞嗞"噪音；如最终装机后噪音不可接受，**软件层将 TIM2 PSC/ARR 改为 PSC=3/ARR=839 即可切回 25 kHz**，硬件无需任何变更。

水泵 + 端在 OUT+ 处**外加一颗 SS54**（与板载 SS54 并联）作冗余续流，**贴近水泵本体**安装（防止长线感性反弹耦合进 MOS 漏极）。LR7843 漏极额外加 **SMBJ24A TVS** 钳位，作为续流二极管失效后的最后防线。

6.3 SSD1315 OLED + 4 按键模块（合一模组）

模块形态：0.96" OLED + 独立按键板组合，OLED 接口 4 脚 I²C，按键板 4 脚独立 GPIO + 1 公共地。

| 引脚 | 模块端 | 接至 STM32 |
|---|---|---|
| OLED `VCC` | 3.3V | 核心板 `3.3V` |
| OLED `GND` | GND | `GND_ISO` |
| OLED `SCL` | I²C SCL | **PB6**（I²C1_SCL） |
| OLED `SDA` | I²C SDA | **PB7**（I²C1_SDA） |
| 按键 `K1` | 一端到 GND，一端到 K1 输入 | **PA1** |
| 按键 `K2` | 同上 | **PA2** |
| 按键 `K3` | 同上 | **PA3** |
| 按键 `K4` | 同上 | **PA4** |
| 按键 `GND` | 共地 | `GND_ISO` |

I²C 总线特性：
  · OLED 默认地址 **0x3C**（7-bit）/ 0x78（8-bit 写）；
  · I²C1 工作于 **Fast-Mode 400 kHz**；
  · **总线上拉**：OLED 模块板载 ~4.7 kΩ 上拉，AT24C64 模块板载 10 kΩ 上拉，**两模块并联后等效 ~3.2 kΩ**，正好落在 Fast-Mode 推荐区间（2.2~4.7 kΩ）→ **不需要再外加任何上拉电阻**；
  · OLED 与 AT24C64 地址不冲突（0x3C vs 0x50），可安全共用一条 I²C1。

按键硬件去抖：
  · 模块自带 10 kΩ 上拉 + 0.1µF 滤波电容；
  · 固件再做 **20 ms 软件扫描去抖**（与总设 § 8 一致）；
  · 4 路按键**优先用独立 GPIO**（已确认 PA4 可用，前提是 W25Qxx 不焊）。

按键功能映射（与总设 § 3.2 完全一致）：

| 物理键 | GPIO | 功能 |
|---|---|---|
| K1 | PA1 | 立即启动 / 跳过自检倒计时 |
| K2 | PA2 | 跳过当前路 / 下一项 |
| K3 | PA3 | 暂停 / 恢复（暂停时立即关 CH5 与全部水阀） |
| K4 | PA4 | 进入参数菜单（每路时长、PWM 起止占空比、阶梯档数） |

OLED 烧屏防护：每路浇灌期间动态翻转 2px 显示坐标；息屏 30s 后进入低对比度待机画面（软件实现，硬件无须处理）。

6.4 AT24C64 配置存储

模块形态：**AT24Cxx 系列通用 EEPROM 小板**（PCB 丝印 `AT24CXX`，1.9 cm × 1.1 cm，4 脚 2.54 mm 排针）。卖家按订单焊接具体容量的芯片；**本项目下单时务必指定焊 `AT24C64`**，并在到货后核对芯片表面丝印。

| 项 | 规格 |
|---|---|
| 型号 | **AT24C64**（64 Kbit = 8 KByte EEPROM） |
| 封装 | 板载 SOIC-8 芯片 + 4 脚 2.54 mm 排针 |
| 接口 | I²C（与 OLED 共用 PB6/PB7） |
| 7-bit 地址 | **0x50**（板内 A0/A1/A2 已全部接 GND） |
| 8-bit 写/读 | 0xA0 / 0xA1 |
| 页大小 | 32 字节 |
| 工作电压 | 1.8V~5.5V，**直接接 3.3V** |
| 写周期 | 5 ms 典型（页写后须等待 ACK Polling） |
| 寿命 | 100 万次擦写 @ 25°C |
| 板载上拉 | SCL/SDA 各内置 **10 kΩ → VCC**（与 OLED 模块的 4.7 kΩ 并联，等效 ~3.2 kΩ，恰好落在 Fast-Mode 推荐区间） |

接线（**仅 4 根线**，A0/A1/A2/WP 已在模块内部处理）：

| 排针引脚 | 模块端 | 接至 STM32 |
|---|---|---|
| 1 | `VCC` | 核心板 `3.3V` + 板上 0.1 µF MLCC 去耦 |
| 2 | `GND` | `GND_ISO` |
| 3 | `SCL` | **PB6**（I²C1_SCL，与 OLED 并联） |
| 4 | `SDA` | **PB7**（I²C1_SDA，与 OLED 并联） |

工艺要点（**必做**）：

1. **WP 写保护脚加固跳线**（**强烈推荐**）：
   原厂模块的 **Pin 7 (WP)** 在板内**悬空**（既不接 GND 也不接 VCC）。AT24C64 内部虽有弱下拉默认允许写入，但在高湿/静电环境下可能被耦合到高电平导致写入失败。**建议在板子背面用一段细漆包线或锡桥把芯片 Pin 7 短接到 Pin 4 (GND)**（或者直接桥到排针的 GND 脚）。30 秒工作量，从此 WP 状态永久确定 = 允许写入。
2. 上拉电阻**不要再外加**：板载 10 kΩ + OLED 模块 4.7 kΩ 并联已是 3.2 kΩ，再加会使总线驱动过强。
3. 与 OLED 模块**走同一条 I²C 总线**：两个 4 脚排针并联到 PB6/PB7，**走线尽量短且绞合**（手工拧麻花）。

存储分区建议（软件层定义，硬件层只关心容量）：

| 起始地址 | 长度 | 内容 |
|---|---|---|
| 0x0000 | 64 B | 魔数 + 版本号 + CRC 头 |
| 0x0040 | 256 B | 用户配置（4 路时长、PWM 起止占空比、阶梯档数、按键灵敏度…） |
| 0x0140 | 128 B | 冗余配置副本（双 Bank 防写入掉电） |
| 0x0200 | 6 KB | 任务日志循环缓冲区（最近 ~200 次浇灌结果） |
| 0x1A00 | 1.5 KB | 预留 |

> AT24C64 取代了总设 V1.2 § 3.2 中的"内部 Flash 末页 EEPROM emulation"。原因：
> · F401 内部 Flash 没有专门 EEPROM 区块，emulation 方案需牺牲整页（最小 16 KB）；
> · 配置参数写入频率虽低（菜单调整时），但外置 EEPROM 寿命/可维护性更优；
> · I²C 已为 OLED 拉好，复用零成本。

7.3 LM2596 模块的工艺要点

· 出厂电位器初始位置不可信，**必须在空载下调到 5.00V ±0.05V**，再用 502 胶 / 滴蜡封固螺丝防震；
· 模块输入并 100 µF/25V 电解 + 1 µF MLCC，输出再并 100 µF/16V 电解 + 0.1 µF MLCC；
· 模块输出**不**与 5V_ISO 短接；两者仅在万用板上各自走一条独立母线。

7.4 B1205S-2WR3 的工艺要点

· 模块底部 PCB 上的覆铜（如有）用美工刀刻断，强制初次级 ≥ 5 mm 物理沟槽；
· 二次侧（5V_ISO）输出端紧贴模块焊 10 µF MLCC，再串 10 µH（饱和电流 ≥ 500 mA）大电流电感，再并 47 µF 电解 —— 形成 π 型滤波；
· 二次侧并 **470Ω 1/4W 色环电阻**作常驻假负载（确保空载电压稳定锁 5.0V，B1205S 未隔载时会上飘到 ≥ 5.5V）；
· 在 MCU VDD 引脚最末端再加 **0.1 µF MLCC** 紧贴芯片焊盘。

布局原则：

12. 与总体设计书 V1.2 的差异说明 (Deltas from V1.2)

| 章节 | V1.2 总设描述 | 本硬件方案 V1.0 | 说明 |

13. 验收测试要点 (Acceptance Tests)

硬件层验收前置（**先于装水试机**）：

| 序 | 测项 | 方法 | 通过判据 |
|---|---|---|---|
| 1 | 双地隔离 | 万用表蜂鸣档测 `GND_DIRTY` ↔ `GND_ISO` | 必须**不通**（断路） |
| 2 | 5V_DIRTY 电压 | LM2596 输出空载 | 5.00V ±0.05V |
| 3 | 5V_ISO 电压 | B1205S 二次侧 + 假负载 | 5.0V ±0.1V |
| 4 | 3.3V 电压 | 核心板 3V3 排针 | 3.30V ±0.05V |
| 5 | I²C 扫描 | MCU 跑测试程序 i2cdetect；**先确认 AT24C64 WP 跳线已焊** | 同时检测到 0x3C 与 0x50；连续写 → 断电 → 读回数据不丢 |
| 6 | 按键 4 路 | 单按测试 | K1~K4 状态翻转无串扰 |
| 7 | 继电器 6 路 | 单路逐一吸合（仅信号侧，水泵未接） | 听到清脆"咔"声，万用表测 NO-COM 导通 |
| 8 | PWM 1kHz | 示波器测 PA0 与 LR7843 OUT+ | 频率 1.000 kHz ±5%，占空比 0~100% 线性可调 |
| 9 | CH5 锁定 | 把 `LO_LEVEL_SHORT` 跳线拔掉模拟"低水位" | CH5 不论 PB1 何电平均无法吸合 |
| 10 | 看门狗复位 | 故意停喂狗 | 256 ms 内 MCU 复位，CH5 默认释放 |

通过以上 10 项后，方可装水、闭环试机。
