# FloraMate 串口调试指令说明 V1.0

## 1. 串口参数

| 项目 | 值 |
|---|---|
| 接口 | `USART1`，PA9 TX / PA10 RX |
| 波特率 | 115200 |
| 格式 | 8N1，无流控 |
| 行结束 | `\r`、`\n`、`\r\n`，或 **USART IDLE**（一帧发送结束，约 1 字符时间无新数据）自动提交 |

## 2. 上电行为

上电后进入 **10 秒串口等待窗**（`APP_SERIAL_DEBUG_BOOT_WAIT_MS`，见 `app_serial_debug_config.h`）：

| 情况 | 行为 |
|---|---|
| 10s 内收到 `debug` / `debug on` / `dbg` | 进入**手动-串口**；OLED 显示命令与结果；**不再打印心跳** |
| 10s 内按下 K1/K2 暂停倒计时，**K1+K2 同时按住 ≥2.5s** | 进入**手动-选择**页（串口/按键/返回上电） |
| 10s 内无上述操作 | **自动模式**：`BOOT → SELFTEST → IDLE_3S → AUTO_RUN` |
| 自动浇灌中按 **K3 停止** | 停止输出并进入**手动-选择**页 |
| 手动-串口 **K4 长按 2s** | 返回 **Manual 选择页** |
| 手动-串口发送 `debug off` / `exit` / `quit` | 回到 Boot Wait；再次超时则走自动模式 |
| 编译宏 `APP_SERIAL_DEBUG_AUTO_ENTER_ON_BOOT=1` | **跳过等待窗**，上电直接进入串口调试态（联调用） |

等待窗内串口每秒输出心跳（示例）：

```text
[I] hb uptime_ms=1000 wait_remain_ms=2000
```

进入调试后串口会打印 `[I] serial debug active (no hb)` 与 `help`，此后无心跳。

OLED：

- **Serial Wait**：倒计时 `debug in Ns`，提示发送 `debug` / `dbg`
- **Serial Debug**：`> 当前命令`、`OK:` / `ERR:` 结果、PWM 与 6 路继电器状态位
- **Key Test**（`key test` 后）：四行 `K1 UP` / `K2 DOWN` / `K3 OK` / `K4 BACK`，按下显示 `ON`（反白），松开为 `--`；发送 `key test off` 退出

## 3. 编译开关

文件：`flora_mate/code/App/serial_debug/app_serial_debug_config.h`

```c
#define APP_SERIAL_DEBUG_BOOT_WAIT_MS (10000U)  /* 等待进入手动的秒数窗口 */
#define APP_MODE_MANUAL_COMBO_HOLD_MS (2500U)   /* K1+K2 进入手动选择 */
#define APP_SERIAL_DEBUG_K4_EXIT_HOLD_MS (2000U) /* 串口手动 K4 长按退出 */

#ifndef APP_SERIAL_DEBUG_AUTO_ENTER_ON_BOOT
#define APP_SERIAL_DEBUG_AUTO_ENTER_ON_BOOT (0) /* 1=上电自动进 debug */
#endif
```

## 4. 命令总览

| 命令 | 说明 |
|---|---|
| `debug` / `debug on` / `dbg` | 进入手动-串口（等待窗内或已调试态均可） |
| `debug off` / `exit` / `quit` / `manual exit` | 退出手动-串口，回到 Boot Wait（等待窗重计；超时走自动） |
| `help` / `?` | 打印命令帮助 |
| `ver` | 打印固件版本和编译时间 |
| `status` | 打印主状态、PWM、继电器、UART 计数 |
| `key test` / `keytest` | 进入按键测试：OLED 实时显示 K1~K4 是否按下；串口在状态变化时打印 `keys k1=...` |
| `key test off` / `key test exit` / `keytest off` | 退出按键测试，回到普通串口调试界面 |
| `pump <0-100>` | 吸合 **CH1** 水泵总电源后设置 PWM；须至少一路阀（CH2~CH5）已开 |
| `pump off` | 水泵 PWM 置 0，并释放 **CH1** |
| `valve open <1-6>` | 打开指定继电器，**不关闭其它路**（可多路同时 on） |
| `valve close` | 关闭全部 6 路继电器 |
| `valve close <1-6>` | 关闭指定继电器 |
| `cfg dump` | 将当前配置打印到日志 |
| `cfg get <name>` | 读取配置字段 |
| `cfg set <name> <value>` | 修改 RAM 中的配置字段 |
| `cfg save` / `cfg load` / `cfg reset` | 保存、重载、恢复出厂配置 |
| `stop` | 立即关 PWM 并释放全部继电器 |
| `run` | 预留；本阶段不启动自动浇灌 |
| `reset` | MCU 软件复位 |

### 继电器用户编号（`valve` 命令，与模块丝印 CH1~CH6 一致）

| 编号 | 模块通道 | 说明 |
|---|---|---|
| 1 | CH1 | 水泵总电源（仅控制水泵供电） |
| 2~5 | CH2~CH5 | 四路电磁阀 |
| 6 | CH6 | 备用 |

`pump` 命令控制 PWM 调速，并会吸合 **CH1** 水泵总电源（正常模式仍要求至少一路阀已开）。

调试模式下 `valve open/close` 走 `Bsp_Relay_DebugSet()`，**无互锁**，可同时打开多路。

## 5. 常用联调顺序

```text
debug
help
key test
key test off
status
valve open 1
valve open 2
pump 30
pump off
valve close
status
```

安全说明：`pump <非零>` 会吸合 **CH1** 并检查至少一路阀（CH2~CH5）已打开；否则返回互锁错误。联调建议先 `valve open 2`（任一路阀）再 `pump`。
