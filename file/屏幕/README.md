# 1.3 寸 OLED 屏幕资料

本目录存放 **1.3 寸 I²C OLED** 模块资料（原理图、初始化说明、厂家例程等）。

## 固件约定（FloraMate V1.0）

| 项目 | 值 |
|------|-----|
| 尺寸 | 1.3 寸 |
| 分辨率 | **128 × 64** |
| 驱动 IC | **SH1106**（常见）；少数为 SSD1306 / SSD1315 |
| 接口 | I²C，7-bit 地址 **0x3C** |
| 与 EEPROM | 共用 MCU **I²C1**（PB6/PB7） |

## 驱动配置（改这里即可，无需动刷屏逻辑）

文件：`code/Bsp/oled/bsp_oled_config.h`

| 宏 | 默认 | 说明 |
|----|------|------|
| `BSP_OLED_USE_SH1106` | `1` | `1`=SH1106（推荐）；`0`=SSD1306/1315 |
| `BSP_OLED_COL_OFFSET` | `2`（SH1106 时） | 列偏移。画面左右错位时试 `0` 或 `4` |

实现：`code/Bsp/oled/bsp_oled.c`

- **SH1106**：按页写显存，每页前设列地址 `0x00\|offset` 与 `0x10\|(offset>>4)`（与 Raspberry Pi `sh1106.py` 一致）。
- **SSD1306**：页寻址模式 `0x20,0x02`，用 `0x21/0x22` 设列页窗口后写 128 字节。

## 现象与排查

| 现象 | 处理 |
|------|------|
| 花屏、斜纹、内容错位 | 确认 `BSP_OLED_USE_SH1106=1`，`COL_OFFSET=2` |
| 整体左右偏 2 像素 | `COL_OFFSET` 改为 `0` 或 `4` |
| 上下颠倒 | 在 `bsp_oled.c` 的 init 中交换 `0xA0/0xA1` 与 `0xC0/0xC8` |
| 全黑但 I²C 在线 | 查 3.3V 供电、SCL/SDA、地址 0x3C |
| 仍为 SSD1306 模块 | `BSP_OLED_USE_SH1106` 设为 `0` 后重新编译 |

## 接线

与 `doc/硬件设计方案_V1.0.md` § OLED 一致：VCC 3.3V、GND、SCL、SDA。

若模块带 **RES** 引脚：上电前拉低 10ms 再拉高（本版 PCB 若 RES 接 3.3V 则无需软件复位）。
